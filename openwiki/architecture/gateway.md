# Gateway Architecture

Aisphere Kernel uses **Envoy Gateway** as the unified edge gateway. All route configuration is **generated from proto contracts** — no hand-written gateway configuration.

## Route Generation Pipeline

```
Proto Contract
  → google.api.http (HTTP binding)
  → aisphere.access.v1.policy (access policy)
  → protoc-gen-go-gateway (code generator)
  → <Service>GatewayManifest() (generated Go function)
  → serverx.RegisterGatewayRoutes(...)
  → RouteRegistry
  → Envoy Gateway HTTPRoute (K8s CRD)
```

## Route Publication Policy

A route is published to the gateway only when **both** `google.api.http` AND `aisphere.access.v1.policy` are declared on the RPC.

### GatewayPublish (tri-state)

| Value | Behavior |
|---|---|
| `UNSPECIFIED` | Generator decides based on policy |
| `ENABLED` | Force publish |
| `DISABLED` | Force skip |

### Gateway Profiles

| Profile | Includes |
|---|---|
| `PublicRouteFilter` | PUBLIC, AUTHENTICATED, AUTHORIZED routes |
| `InternalRouteFilter` | Service-to-service routes |

### Route Type Decision Table

| Route Type | Public Gateway | Internal Gateway | Direct |

## Auth Mode → SecurityPolicy Mapping

| Proto Policy | Gateway SecurityPolicy | Backend Behavior |
|---|---|---|
| `AUTH_NONE` / PUBLIC | None | No principal required |
| `AUTHENTICATED` / `AUTHN_ONLY` | OIDC/JWT | Principal required, no authz |
| `AUTHORIZED` / `AUTHZ` | OIDC/JWT | Principal + accessx authz |
| `INTERNAL` | No public route | Internal service token |

**Key rules**:
- No global OIDC SecurityPolicy — each app uses independent SecurityPolicy
- All routes must sanitize headers (strip `x-aisphere-*` and `x-internal-jwt` from client input)
- `claimToHeaders` only outputs `x-aisphere-external-*` headers

## Envoy Gateway Responsibilities

- Edge routing (HTTP → gRPC transcoding)
- OIDC/JWT verification via OIDC Discovery + JWKS
- Header sanitization (strip spoofable headers)
- Claim-to-header injection (`x-aisphere-external-*`)
- Ingress timeout, rate limiting, trace propagation

## Envoy Gateway Does NOT Do

- Resource-level authorization (that's `accessx` + `authz` + SpiceDB)
- Business database queries
- IAM/Authz service replacement

## Three-Stage Verification Flow

1. **Unauthenticated** → 302 redirect (browser) or 401 (API)
2. **Authenticated but unauthorized** → 403
3. **Authenticated and authorized** → 200 + audit

## Gateway Runtime (gatewayx)

The `gatewayx` package provides runtime building blocks for gateway/BFF services:

| Component | Purpose | Source |
|---|---|---|
| `Route` | A single gateway route definition | `/gatewayx/routes.go` |
| `Manifest` | Generated route manifest for a service | `/gatewayx/manifest.go` |
| `RouteRegistry` | Route registration and lookup | `/gatewayx/registry.go` |
| `Dispatcher` | Route dispatching | `/gatewayx/dispatcher.go` |
| `Matcher` | Route matching | `/gatewayx/matcher.go` |
| `OperationInvoker` | Invoke backend operations | `/gatewayx/invoker.go` |
| `ManifestPublisher` | Publish manifests to a registry | `/gatewayx/publisher.go` |
| `InternalJWT` | Internal service token generation/verification | `/gatewayx/internaljwt.go` |
| `Session` | Gateway session management | `/gatewayx/session.go` |
| `ProxyConfig` | Proxy configuration | `/gatewayx/proxy.go` |

## Route Publication

The `serverx` package manages route publication:

```go
// Generated by protoc-gen-go-gateway
func (s *TodoService) GatewayManifest() *gatewayx.Manifest {
    return &gatewayx.Manifest{
        Routes: []gatewayx.Route{...},
    }
}

// Registration in serverx
serverx.RegisterGatewayRoutes(manifest, registry)
```

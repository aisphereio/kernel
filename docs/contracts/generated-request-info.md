# Generated RequestInfo Resolver Contract

Kernel follows the Kubernetes apiserver pattern of resolving request metadata once and sharing it across the whole request chain.

## Rule

Every proto service that declares `aisphere.access.v1.policy` and participates in authz generation should produce:

```go
func <Service>RequestInfoResolver(ctx context.Context, operation string, req any) (requestx.Info, bool, error)
```

The resolver is generated by `protoc-gen-go-authz` and must be wired through:

```go
autowire.WithRequestInfoResolver(v1.TodoServiceRequestInfoResolver)
autowire.WithClientRequestInfoResolver(v1.TodoServiceRequestInfoResolver)
```

## Middleware contract

Runtime middleware must read request metadata from `requestx.Info` first. Transport operation strings are fallback only.

Required consumers:

- authn enrichment
- authz/access resolver
- audit
- rate limit key generation
- metrics labels
- logs
- tracing spans
- errorx response metadata

## Agent rule

Agents must not parse raw HTTP paths, gRPC full method names, or operation strings inside business logic. Add missing metadata to proto options or the generator, then consume `requestx.Info`.
